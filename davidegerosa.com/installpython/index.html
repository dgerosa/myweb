The best way to install Python

Python is cool, especially if you a scientist. All sort of scientific algorithms are already there (written and debugged!), you just have to use them.

This is a step-by-step guide to the best way I found to install and use python for science, somehow taken from this stack overflow question. Instructions here are given for mac OSX  but can be easily generalized to any unix system (e.g. replacing curl with wget and so on…). I will also stay with python 2, as everything still works (see here for more information on the issue).

Local is better: install a new python distribution

Most (if not all) unix systems already come with a python distribution installed. However, it is advisable to install a local distribution and do your scientific stuff from there… If you screw something up, you can just delete everything and your OS is safe.

First, go to  this URL:

https://www.python.org/ftp/python/

and check what is the latest version of python 2. At the time of writing, this is 2.7.10, change the following lines if you want a different version. I am going to assume you want to put your new python distribution in a directory called “localpython” in your home directory

cd
mkdir localpython
cd localpython
curl  https://www.python.org/ftp/python/2.7.10/Python-2.7.10.tgz > Python-2.7.10.tgz
tar -zxvf Python-2.7.10.tgz
cd Python-2.7.10

# "make clean" may be necessary here for earlier versions
./configure --prefix=${HOME}/localpython
make
make install

If, at any time, you need to start again, just

rm -rf ${HOME}/localpython

Different boxes: python’s virtualenv

Python’s virtual environments are kind of separate boxes, where you can install modules and packages locally. You can have different boxes for different projects, or a single box for all your python stuff. Again, if you screw something up, you can just delete the box and start again.

First, go to this URL:

https://pypi.python.org/pypi/virtualenv

and check what is the latest version of virtualenv. At the time of writing, this is 13.1.2, change the following lines if you want a different version.

cd ${HOME}/localpython
curl  https://pypi.python.org/packages/source/v/virtualenv/virtualenv-13.1.2.tar.gz > virtualenv-13.1.2.tar.gz
tar -zxvf virtualenv-13.1.2.tar.gz
cd virtualenv-13.1.2
${HOME}/localpython/bin/python setup.py install

In the box: create a virtualenv using your new python distribution

Now we create a virtual environments specifying it should run python from our local distribution. Let’s call our virtual environment “box” and place it in localpython

cd ${HOME}/localpython 
${HOME}/localpython/bin/virtualenv box --python=${HOME}/localpython/bin/python

If you get an error involving zlib, this is likely to be related to an upgrade of the OS: check here for a solution using homebrew. You have created the box, but you’re still out of it. To get into the box

chmod u+x ${HOME}/localpython/box/bin/activate
source ${HOME}/localpython/box/bin/activate

Now you’re in the box, and you should see “(box)” close to your username in the terminal. You can test it with

which python
>>>  ${HOME}/localpython/box/bin/python

If you enter a python console, you should get today’s date and time (or yesterday’s if you found these instructions exhausting)

python
>>>  Python 2.7.10 (default, TODAY! )

To get out of the box, type

deactivate

and the label “(box)” should disappear from your terminal.

You may want to add this last command in your .bashrc

echo "" >>${HOME}/.bashrc
echo "alias inthebox='source ${HOME}/localpython/box/bin/activate' " >> ${HOME}/.bashrc
source ${HOME}/.bashrc

Now type “inthebox” and “deactivate” to get in and out of your new virtual environment.

Science is fun again: install packages

Now you’re done. Virtualenv comes with pip, the tool to install python modules from the Python Package Index PyPi. From within your box, to install a package type

pip install PACKAGE_NAME

First of all, upgrade pip itself. It’s not needed strictly, but my experience is that it may fix issues, especially on mac OS X

pip install --upgrade setuptools pip

You can try with numpy, scipy, matploltib, and my own precession module to study black holes.

Another way

Another great way of safely using python of a MAC is homebrew. Homebrew installs a new version of python (by default the latest 2.x version available) and set is as default

brew install python

You can see where it has been installed typing

which python
# for me this is /usr/local/bin/python

and then create a virtual environment using that python executable

virtualenv box --python=/usr/local/bin/python
