Migrate SVN to GIT

This is a supershort guide to migrate old SVN repositories to GIT. I assume you have very simple SVN repositories (one single branch, no tags) like mine. If you have a more complicated structure, have a look here and here.

We need the git-svn tool. Install it if you don’t have it.

First, enter some directory in your local machine (e.g. where you keep all your geek stuff on GIT, say repositories).

cd ~/repositories

Now, clone your old SNV repository using git svn. Of course, you need access to it: in the example below I use the svn+ssh protocol, but you may have http here depending on your setup (just to be clear, this is the same link you used to checkout the SVN repo)

git svn clone svn+ssh://USER@HOST/PATH/SVNNAME

If you want a remote host (and you do), now you need to create a repository on some online service. I will use bitbucket in the example below (which gives free unlimited everything to scientists), but github is also a popular choice. Assuming you have created a repository using the bitbucket website,  now add it and push.

cd SVNNAME
git remote add origin git@bitbucket.org:BB_USERNAME/GITNAME
git push origin --all
